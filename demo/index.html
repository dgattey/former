Former beta

A project by Dylan Gattey available on Github

Sick of losing everything in a form because you accidentally navigated away? Wish you could go back to a previous version of something you've typed? Former is here to rescue you with automatic, timeline-based backups, saved right to your browser.

How it Works

Former works by injecting some simple Javascript code into the page, dynamically changing every input field into a Former-ified field. Every textarea and input gets two buttons that appear when you hover over the element. Click the history button, then scrub through the past with ease and speed. Clear all history for the field with the clear button, in case some things are better left in the past.

Even if your browser crashes, or your internet goes out, or you feel like refreshing the page, your content will be there. Former's developers never see your content, as it's saved in local, browser-based storage. Your content is safe with you. See it in action in the demo below!

Compatibility

Former uses some new HTML5 technologies, and thus supports only the newest browsers. You should be able to run it on Chrome 22+, Safari 7+, IE 9+, and Firefox 18+, and perhaps some older browsers that support the necessary standards.

Technical Details

Former works by saving every keystroke to the browser's localStorage cache, and exposing that history to the user dynamically with the history button. Clearing simply deletes localStorage's cache. If the change made at any point is the same as the last version, Former saves your data just once.

Installation Coming soon

Former will soon be packaged as a Chrome and Safari extension so you can use it on any site by simply installing a plugin. For now, if you'd like to use Former yourself, you'll need to include former.js and former.css from this website or the Github repo. Additionally, the following are requirements for full functionality: jQuery 2+ and Tipped.
